# 工具集

Spider 内置了四个工具模块，涵盖 Shell 执行、文件读写、文档转换和自开发能力。

---

## Shell 工具

**文件**: `tools/shell.py` | **函数**: `execute_shell()`

执行 shell 命令并返回输出（stdout + stderr）。

### 定义

```python
async def execute_shell(command: str, timeout: int = 60) -> str:
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `command` | `string` | — | 要执行的 shell 命令 |
| `timeout` | `integer` | 60 | 超时秒数 |

### Schema

```python
SHELL_TOOL_SCHEMA = {
    "type": "object",
    "properties": {
        "command": {
            "type": "string",
            "description": "要执行的 shell 命令",
        },
        "timeout": {
            "type": "integer",
            "description": "超时秒数（默认 60）",
            "default": 60,
        },
    },
    "required": ["command"],
}
```

### 使用示例

```python
from tools.shell import execute_shell
result = await execute_shell("ls -la")
print(result)
```

通过 Agent：

```python
agent.register_tool("shell", "执行 shell 命令", execute_shell, SHELL_TOOL_SCHEMA)
result = await agent.run("查看当前目录文件")
```

### 错误处理

| 场景 | 返回值 |
|------|--------|
| 空命令 | `Error: empty command` |
| 命令超时 | `Error: command timed out after 60s` |
| 命令不存在 | `Error: command not found - ...` |

### 安全说明

- 命令以当前用户权限执行
- 有 60 秒超时保护，防止命令挂死
- 超时后进程会被 kill

## 文件读写工具

**文件**: `tools/read_write.py` | **函数**: `read_file()`, `write_file()`, `list_files()`

### read_file

读取文件内容，支持行号范围控制。

```python
async def read_file(path: str, offset: int = 0, limit: int = 2000) -> str:
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `path` | `string` | — | 文件路径（支持 `~` 展开） |
| `offset` | `integer` | 0 | 起始行号 |
| `limit` | `integer` | 2000 | 最大读取行数 |

**Schema：**

```python
READ_FILE_SCHEMA = {
    "type": "object",
    "properties": {
        "path": {"type": "string", "description": "文件路径"},
        "offset": {"type": "integer", "description": "起始行号", "default": 0},
        "limit": {"type": "integer", "description": "最大行数", "default": 2000},
    },
    "required": ["path"],
}
```

### write_file

写入文件（覆盖已有内容），自动创建父目录。

```python
async def write_file(path: str, content: str) -> str:
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `path` | `string` | 文件路径 |
| `content` | `string` | 要写入的内容 |

返回：`✅ 已写入 {path} ({n} 字符)`

**Schema：**

```python
WRITE_FILE_SCHEMA = {
    "type": "object",
    "properties": {
        "path": {"type": "string", "description": "文件路径"},
        "content": {"type": "string", "description": "文件内容"},
    },
    "required": ["path", "content"],
}
```

### list_files

列出目录内容，显示文件和子目录。

```python
async def list_files(path: str = ".") -> str:
```

返回格式：

```
📁 /path/to/dir/
  📁 subdir/
  📄 file.txt  (1234 bytes)
  📄 script.py  (567 bytes)
```

**Schema：**

```python
LIST_FILES_SCHEMA = {
    "type": "object",
    "properties": {
        "path": {"type": "string", "description": "目录路径", "default": "."},
    },
}
```

### 错误处理

| 场景 | 返回值 |
|------|--------|
| 文件不存在 | `Error: file not found: {path}` |
| 路径是目录 | `Error: not a file: {path}` |
| 目录不存在 | `Error: path not found: {path}` |
| 写入权限不足 | `Error writing file: ...` |

## 文档转换工具

**文件**: `tools/convert.py` | **函数**: `docx_to_pdf()`, `pdf_to_docx()`

### 功能

- Word (.docx) → PDF
- PDF → Word (.docx)

### 自动后端选择

工具自动检测可用的转换后端，优先级：

| 优先级 | 后端 | 安装方式 | 转换质量 |
|--------|------|---------|---------|
| 1 | LibreOffice | `brew install libreoffice` | :star: :star: :star: 保留排版 |
| 2 | Python 库 | `pip install python-docx PyMuPDF fpdf2` | :star: :star: 纯文本 |

### 函数

```python
async def docx_to_pdf(input_path: str, output_path: str = None) -> str:
    """Word 转 PDF"""

async def pdf_to_docx(input_path: str, output_path: str = None) -> str:
    """PDF 转 Word"""
```

### Schema

```python
DOCX_TO_PDF_SCHEMA = {
    "properties": {
        "input_path": {"type": "string", "description": "输入的 Word 文件路径"},
        "output_path": {"type": "string", "description": "输出的 PDF 路径（可选）"},
    },
    "required": ["input_path"],
}
```

### 错误处理

| 场景 | 提示 |
|------|------|
| 文件不存在 | 文件不存在 |
| 格式不支持 | 需要 .docx / .pdf 格式 |
| 无可用后端 | 提示安装 LibreOffice 或 Python 库 |

## 自开发工具

**文件**: `tools/self_dev.py`

Spider 具备自我修改代码的能力——通过以下 6 个工具，Agent 可以探索、审查、修改自己的源码并提交更改。

### self_map — 项目结构导航

```python
async def self_map(target: str = "") -> str:
    """查看项目结构和模块职责"""
```

查看整个项目的目录树，或定位特定模块的位置。

**Schema：**

```python
SELF_MAP_SCHEMA = {
    "properties": {
        "target": {"type": "string", "description": "目标模块名（为空则显示全部）"},
    },
}
```

### self_find — 源码搜索

```python
async def self_find(pattern: str, file_pattern: str = "*.py") -> str:
    """语义搜索源码，找到函数/类/变量位置"""
```

类似 grep 的语义版，根据描述找到代码位置。

### self_validate — 语法验证

```python
async def self_validate(target: str = ".") -> str:
    """AST 语法检查，确保修改后的代码没错"""
```

使用 Python AST 解析器检查语法，不执行代码。

### self_review — 代码审查

```python
async def self_review() -> str:
    """查看当前 git diff，审查改动"""
```

显示未暂存的修改，帮助 Agent 在提交前检查。

### self_edit — 安全修改

```python
async def self_edit(path: str, old_string: str, new_string: str) -> str:
    """安全修改代码：自动备份→替换→语法验证→失败回滚"""
```

这是最关键的自开发工具，实现了安全的修改流程：

1. **备份** — 修改前自动备份原始文件到 `.backup/`
2. **替换** — 执行字符串替换
3. **验证** — 用 `self_validate` 检查语法
4. **回滚** — 如果验证失败，自动恢复备份

### self_commit — 自动提交

```python
async def self_commit(message: str) -> str:
    """自动 git add + commit"""
```

将当前修改提交到 git，自动处理暂存操作。

### 工具间协作

这些工具通常按以下流程配合使用：

```
self_map → 了解项目结构
    │
self_find → 定位要修改的代码
    │
self_edit → 安全修改
    │
self_review → 审查改动
    │
self_commit → 提交
```
