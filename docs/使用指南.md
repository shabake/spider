# 使用指南

## 三种启动模式

### Web UI :globe_with_meridians:

适合可视化操作，实时看到 Agent 的思考和工具调用过程。

```bash
python main.py --web
# 可选参数
python main.py --web --port 8080 --host 0.0.0.0
```

打开浏览器访问 `http://127.0.0.1:8888`，你会看到：

- **聊天界面** — 输入任务，实时流式显示回复（Markdown 渲染 + 代码高亮）
- **工具调用卡片** — 每个工具调用都可展开查看参数和结果
- **对话历史** — 侧边栏列出所有历史对话，可点击回顾
- **快捷建议** — 空状态时显示常用任务快捷按钮
- **深色主题** — Claude 风格，护眼

### 交互式 CLI :speech_balloon:

适合连续对话，反复问问题。

```bash
python main.py -i
```

进入后看到 `>` 提示符，直接输入任务即可：

```
> 帮我看看当前目录
> 检查磁盘空间
> /tools          ← 查看可用工具
> /help           ← 查看帮助
> quit            ← 退出
```

| 命令 | 说明 |
|------|------|
| `quit` / `exit` / `退出` | 退出交互模式 |
| `/tools` | 查看所有可用工具 |
| `/help` | 查看命令帮助 |

### 单次任务 :dart:

适合问一个问题就结束的场景。

```bash
python main.py "任务描述"
```

支持的任务类型：

```bash
# 查文件
python main.py "当前目录有哪些文件？"

# 查系统
python main.py "我的电脑还剩多少磁盘空间？"

# 文件操作
python main.py "在桌面创建一个 hello.txt，写入 Hello Spider"

# 文档转换
python main.py "把 ~/Desktop/test.docx 转成 PDF"

# 委托子任务
python main.py "用 delegate_task 分别查一下系统名称和当前时间"
```

## 传 API Key 的方式

### 方式 1：环境变量（推荐）

```bash
export DEEPSEEK_API_KEY="sk-xxxx"
python main.py "你的任务"
```

### 方式 2：命令行参数

```bash
python main.py --api-key "sk-xxxx" "你的任务"
```

### 方式 3：写入 shell 配置

```bash
echo 'export DEEPSEEK_API_KEY="sk-xxxx"' >> ~/.zshrc
source ~/.zshrc
# 之后直接运行即可
python main.py "你的任务"
```

## 角色配置（Profile） {: #profile }

通过 `--profile` 切换不同角色，每个角色有独立的提示词、工具集和记忆数据库，**互不干扰**。

```bash
# 财务助手（独立记忆）
python main.py -i --profile finance

# 编程助手（独立记忆）
python main.py -i --profile coding
```

### 内置角色

| Profile | 说明 | 工具范围 | 记忆数据库 |
|---------|------|---------|-----------|
| `finance` | 财务规划、记账、投资 | shell、文件读写 | `data/finance.db` |
| `coding` | 代码开发、调试、审查 | shell、文件读写、自开发工具 | `data/coding.db` |

### 自定义 Profile

在 `profiles/` 目录下新建 `.yaml` 文件：

```yaml
# profiles/my-assistant.yaml
name: my-assistant
display: 我的助手 🤖
description: 自定义助手

prompt: |
  你是一个自定义助手...
  （这里写角色的 system prompt）

# 独立记忆数据库（可选，默认使用公共库）
db_path: ./data/my-assistant.db

# 可用工具白名单（只注册这些工具）
tools:
  - shell
  - read_file
  - write_file

# 工具风险覆盖
risk_overrides:
  shell: confirm
```

然后运行：

```bash
python main.py -i --profile my-assistant
```

### 不传 profile

默认行为不变，使用公共记忆库、全部工具。

## 战略推理模式 {: #strategy }

Spider 支持 **StraTA 风格战略推理**，启用后 Agent 会采用更深度的思考流程：

> 推断真实目标 → 制定全局策略 → 识别约束和风险 → 分阶段执行 → 每阶段自查修正 → 输出最优结果

### CLI 启用

```bash
python main.py --strategy "分析这个项目的架构"
# 或交互模式
python main.py --strategy -i
```

### Web UI 启用

```bash
python main.py --web --strategy
```

启用后顶部状态栏显示 `🧠 战略推理`。

### 环境变量

```bash
export SPIDER_STRATEGY=true
python main.py "你的任务"
```

### 适合场景

| 场景 | 说明 |
|------|------|
| 复杂分析 | 多因素决策、架构评审 |
| 商业判断 | 需要权衡成本和收益 |
| 风险敏感 | 执行前需要全面评估 |
| 创意方案 | 需要结构化、可落地的方案设计 |

日常简单任务（查文件、看目录）不需要启用，战略模式会增加思考深度但不影响功能。

## 性能优化

### 工具并行执行

当 Agent 一轮中调用多个工具时，**同时执行**而不是依次执行：

```
# 以前（串行）：tool_A + tool_B = 耗时 2s
# 现在（并行）：tool_A + tool_B = 耗时 1s（取最慢的那个）
```

复杂任务加速 20-40%，无需配置，默认启用。

### 上下文智能压缩

自动管理对话上下文，防止长对话 token 超限：

| 压缩次数 | 策略 | 说明 |
|---------|------|------|
| 第 1 次 | LLM 摘要 | 保留关键信息 |
| 第 2+ 次 | 自动截断 | 节省 token，避免重复调用 LLM |
| 紧急 | 减少保留消息数 | 仍超限时放弃早期消息 |

### 记忆选择性注入

简单任务（少于 30 字）不自动 recall，节省 token。复杂任务自动注入相关记忆。

### 先规划再执行

```bash
python main.py --plan "帮我重构项目结构"
```

Agent 先出执行计划，再按步骤执行，避免多走弯路。复杂任务自动启用。

## 长时间任务 :rocket:

写项目、批量处理等大任务，Agent 默认 30 轮可能不够，组合使用以下参数：

```bash
# 写一个 Web 项目：无限轮数 + 跳过确认
python main.py -y --max-turns 200 "帮我用 FastAPI 写一个博客系统"

# Web UI 模式
python main.py --web -y --max-turns 200

# 交互模式
python main.py -i -y --max-turns 100
```

| 参数 | 作用 |
|------|------|
| `-y` | 跳过所有确认弹窗，不中断 |
| `--max-turns N` | 最大执行轮数（默认 30，大项目设 100-300） |
| `--strategy` | 可选，让 Agent 先规划再执行 |

### 不中断的保障

- **上下文自动压缩**：对话太长时自动摘要早期内容，节省 token
- **持久化记忆**：执行过程和结果会自动保存
- **Escape 随时取消**：按 `⎋` 键随时停，不会卡死

## 人机交互确认 {: #confirm }

某些操作有风险（如执行 shell 命令、写入文件、修改代码），Agent 在执行前会暂停等用户确认。

### 风险等级

| 等级 | 行为 | 示例工具 |
|------|------|----------|
| 🟢 `safe` | 直接执行，无需确认 | `read_file`, `list_files`, `self_find` |
| 🟡 `confirm` | 执行前弹窗/提示确认 | `shell`, `write_file`, `self_edit` |
| 🔴 `dangerous` | 默认禁止，需手动放行 | 预留（高风险操作） |

### CLI 模式确认

启用 confirm 等级的工具时，终端会暂停等待输入：

```
  ⚠️  需要确认 (confirm)
  🛠️  shell(command=rm -rf /tmp/test)
  ▸ 继续执行？(y/N):
```

输入 `y` 或 `yes` 继续，直接回车默认拒绝。

### Web UI 确认

Agent 调用 confirm 工具时，浏览器弹出确认弹窗，点"继续"放行，"取消"拒绝。

### 自定义覆盖风险等级

```python
# 让 write_file 也变成 safe（不确认直接写）
agent.tools.set_risk_override("write_file", "safe")

# 让 read_file 需要确认（虽然不太必要）
agent.tools.set_risk_override("read_file", "confirm")
```

### 跳过确认模式 {: #no-confirm }

如果你信任当前任务，想省去每次确认的步骤，可以关掉确认：

```bash
# CLI 交互模式
python main.py -i -y

# 单次任务
python main.py -y "帮我批量处理文件"

# Web UI
python main.py --web -y

# 或设置环境变量
export SPIDER_NO_CONFIRM=true
python main.py -i
```

关闭后 `safe` 和 `confirm` 等级的工具都直接放行，只有 `dangerous` 级别的仍会拦截。状态栏显示 `⚡ 无确认`。

### 已标记为 confirm 的工具

| 工具 | 风险等级 | 理由 |
|------|----------|------|
| `shell` | 🟡 confirm | 可执行任意命令，影响系统 |
| `write_file` | 🟡 confirm | 会覆盖已有文件 |
| `self_edit` | 🟡 confirm | 修改源码有风险 |
| `self_commit` | 🟡 confirm | 提交到 git 需谨慎 |
| 其他所有工具 | 🟢 safe | 只读或可撤回操作 |

## Agent 内部流程

当你输入一个任务后，Agent 内部经历了这些步骤：

```
你输入
  │
  ├─ 自动匹配技能（如果有匹配的）
  │     └─ 例如 "检查磁盘空间" → 命中 disk-check 技能
  │
  ├─ 自动 recall 相关记忆
  │
  ├─ LLM 思考 → 流式输出
  │     ├─ 决定调用工具 → 执行 → 结果加入上下文 → 继续思考
  │     └─ 决定结束 → 返回最终回复
  │
  └─ 持久化：保存对话 + 记忆
```

## 示例场景

### 场景 1：日常系统检查

```bash
python main.py "检查磁盘空间"
```

自动匹配 `disk-check` 技能 → 执行 `df -h` + `df -hi` → 输出分析报告。

### 场景 2：探索新项目

```bash
python main.py "看看 spider 项目有哪些核心模块，每个文件是干什么的"
```

Agent 会先用 `list_files` 看目录结构，再用 `read_file` 读关键文件。

### 场景 3：保存技能

```bash
python main.py "查一下系统运行时间，然后把方法保存为技能"
```

Agent 执行 `uptime` → 调用 `save_skill` 工具 → 以后再说"查运行时间"就能自动执行。

### 场景 4：子代理并行查询

```bash
python main.py "分别查一下系统版本、磁盘空间、内存使用"
```

主 Agent 创建 3 个子 Agent 并行执行，最后汇总报告。

## MCP 扩展设置

Spider 支持 MCP (Model Context Protocol)，可以连接社区提供的各种工具服务。

### 配置方式

编辑项目根目录的 `mcp_servers.json`：

```json
{
  "servers": [
    {
      "name": "computer-use",
      "description": "电脑控制（鼠标、键盘、截图）",
      "command": "npx",
      "args": ["-y", "computer-use-mcp"],
      "enabled": true
    }
  ]
}
```

### 如何获取 MCP 服务

搜索社区 MCP 服务：[Awesome MCP Servers](https://github.com/wong2/awesome-mcp-servers)

热门服务：
- **computer-use-mcp** — 电脑控制（鼠标/键盘/截图）
- **mac-mcp-server** — macOS 原生控制（44 个工具）
- **Playwright MCP** — 浏览器自动化（微软官方）

### 使用

配置好重启 Spider，MCP 工具会自动出现在工具列表中，工具名以 `mcp_` 开头，LLM 会自动使用它们。

## 步骤技能（升级版）

技能现在可以定义**固定执行步骤**，匹配后自动按顺序执行。

```yaml
# skills/daily-check.yaml
steps:
  - name: 磁盘使用
    tool: shell
    params:
      command: df -h
  - name: 内存使用
    tool: shell
    params:
      command: free -h
prompt: 汇总以上结果
```

步骤执行完后，LLM 会对结果进行分析汇总。

### 让 Agent 自己保存步骤技能

在交互模式或单次任务中，先让 Agent 做一次任务，然后说：

> "把刚才的方法保存为带步骤的技能"

Agent 会自动分析自己的操作记录，生成带步骤的技能文件。

## 常见问题

### Q: 启动报错 "API Key 无效"

API Key 没设置或不对。参考上方"传 API Key 的三种方式"。

### Q: 启动报错 "ModuleNotFoundError"

缺少依赖，运行：

```bash
pip install openai pyyaml rich fastapi uvicorn --break-system-packages
```

### Q: 启动后卡住不动

检查网络能否访问 `api.deepseek.com`。可能需要代理。

```bash
curl https://api.deepseek.com/v1/models -H "Authorization: Bearer $DEEPSEEK_API_KEY"
```

### Q: WARNING 说 Embedding 失败

你的 API Key 没有 DeepSeek Embedding 权限，不影响使用。已自动降级为关键词搜索。

### Q: 对话变慢了怎么办？

Claude Code 对话太长时会变慢，有两种方式解决：

**方式 1：/compact（推荐）**
在输入框输入 `/compact`，Claude Code 会自动压缩对话历史为摘要。

**方式 2：新开会诊**

所有项目资产都保存在本地硬盘，不会丢失。
