# 架构设计

Spider 采用**三层分层架构**：Agent 引擎 → 工具系统 → LLM 接口。

```
┌─────────────────────────────────────────────────────────────┐
│                        Spider                               │
│                                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌────────────┐    │
│  │ Agent   │  │SubAgent │  │ Skill   │  │ Memory     │    │
│  │ Engine  │──│ Pool    │──│ Manager │  │ Store      │    │
│  │ (ReAct) │  │         │  │         │  │(SQLite+    │    │
│  └────┬────┘  └─────────┘  └─────────┘  │ FTS5+Vec)  │    │
│       │                                  └────────────┘    │
│       ▼                                                     │
│  ┌───────────┐  ┌─────────┐  ┌───────────┐                │
│  │ Tool      │  │ Skills  │  │ CLI       │                │
│  │ Registry  │  │ (YAML)  │  │(Rich 终端)│                │
│  │ ├─ shell  │  └─────────┘  └───────────┘                │
│  │ ├─ file   │                                             │
│  │ ├─ convert│  ┌───────────┐                              │
│  │ ├─self_dev│  │ Web UI    │                              │
│  │ └─ ...    │  │(FastAPI + │                              │
│  └─────┬─────┘  │ SSE+HTML) │                              │
│        │        └───────────┘                              │
│        ▼                                                    │

│  │ └─ embeddings       │                                   │
│  └─────────────────────┘                                   │
└─────────────────────────────────────────────────────────────┘
```

## 核心架构模式

### 1. Agent 循环 (ReAct)

Agent 的核心是一个 **思考→行动→观察** 循环：

```
用户任务 → 系统提示 + 对话历史 + 工具定义
              │
              ▼
         ┌──────────┐
         │ LLM 思考  │  ← 自动注入相关记忆 + 匹配技能
         │ (think)   │
         └────┬─────┘
              │
          ┌───┴───┐
          │       │
          ▼       │ 需要工具？
     ┌────────┐   │
     │ 执行工具 │   │
     │ (act)   │   │
     └───┬────┘   │
         │        │
         ▼        │
     ┌────────┐   │
     │ 观察结果 │──┘  ← 工具结果加入对话上下文，继续循环
     └────────┘
          │
          ▼
     返回最终结果
```

### 2. 数据流

```
用户输入
    │
    ▼
┌────────────────────────────────────────────┐
│ Agent.run(task)                            │
│  ┌────────────────────────────────────┐    │
│  │ 1. Memory.recall(task)             │    │
│  │    → 注入相关记忆到 system prompt  │    │
│  │ 2. SkillManager.get_skill_prompts  │    │
│  │    → 注入匹配技能到 system prompt  │    │
│  │ 3. ReAct 循环 (max 30 turns)       │    │
│  │    ├─ LLM.think_stream()           │    │
│  │    ├─ ToolRegistry.execute()       │    │
│  │    └─ 追加结果到 messages          │    │
│  └────────────────────────────────────┘    │
│  → 返回 content                           │
└────────────────────────────────────────────┘
    │
    ▼
输出给用户 (CLI / Web / Platform)
```

### 3. 消息协议

Agent 内部使用 OpenAI 兼容的消息格式：

| 角色 | 用途 |
|------|------|
| `system` | 系统提示词（含技能注入 + 记忆注入） |
| `user` | 用户输入 |
| `assistant` | LLM 回复或 `tool_calls` |
| `tool` | 工具执行结果 |

## 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 编程语言 | Python | 快速迭代，生态丰富 |
| LLM 协议 | OpenAI 兼容 | 可替换多种模型 |
| 工具定义 | JSON Schema | LLM 原生理解 |
| 记忆存储 | SQLite + FTS5 | 零配置，单文件 |
| UI 方案 | FastAPI + SSE | 实时性，简单 |
| 子代理 | 独立 Agent 实例 | 上下文隔离 |
| 技能格式 | YAML | 人类可读写 |
