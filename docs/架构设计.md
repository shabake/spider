## 完整数据流

```
用户输入: "分析项目结构并写README"
    │
    ├── CLI 解析: --profile coding -y --max-turns 50
    │     ├── 加载 Profile → 独立 system prompt + 工具白名单 + 独立 DB
    │     └── 创建 Agent(max_turns=50, confirm_enabled=False)
    │
    ├── Agent.run(task)
    │     │
    │     ├── _setup_messages(task)
    │     │     ├── system = SYSTEM_PROMPT
    │     │     ├── + StraTA 战略提示（如果 --strategy）
    │     │     ├── MemoryStore.recall(task) → 注入相关记忆
    │     │     └── SkillManager.find_matching(task) → 注入技能指引
    │     │
    │     └── ReAct 循环 (for turn in range(max_turns))
    │           │
    │           ├── LLM.think_stream(messages, tools)
    │           │     └── 流式输出思考内容到 CLI/Web
    │           │
    │           ├── is_done? → 返回结果 ✅
    │           │
    │           ├── 遍历所有 tool_calls
    │           │     │
    │           │     ├── 显示工具调用（CLI/Web）
    │           │     │
    │           │     ├── _confirm_tool(tc)  ← 风险检查
    │           │     │     ├── safe → 直接放行
    │           │     │     ├── confirm → 弹窗/输入确认
    │           │     │     │     ├── Web: SSE confirm → 用户点按钮
    │           │     │     │     └── CLI: terminal y/N
    │           │     │     ├── dangerous → 默认拒绝
    │           │     │     └── confirm_enabled=False → 放行
    │           │     │
    │           │     ├── 拒绝 → 记录"用户取消"，继续下一工具
    │           │     │
    │           │     └── 通过 → ToolRegistry.execute(tc)
    │           │           └── 结果追加到 messages
    │           │
    │           └── 上下文超长?
    │                 └── ContextManager.compress() → LLM 摘要压缩
    │
    ├── 持久化
    │     ├── save_message(conversation_id, role, content)
    │     └── update_conversation(turn_count, summary)
    │
    └── 返回最终回复
```

## Profile 系统（--profile）

Profile 实现了**角色隔离**——每个角色有独立的提示词、工具和记忆库：

```
profiles/
  ├── finance.yaml      # 财务助手：shell + web，独立 DB
  ├── coding.yaml       # 编程助手：shell + web + self_dev，独立 DB
  └── (你自定义).yaml    # 任意新增，无需改代码
```

### 隔离级别

| 维度 | 说明 |
|------|------|
| **System Prompt** | 每个角色有独立的角色定义和行为准则 |
| **工具白名单** | 只注册需要的工具，减少误操作 |
| **记忆数据库** | 独立的 SQLite 文件，对话和记忆互不见 |
| **风险覆盖** | 可自定义每个工具的风险等级 |

## 人机交互确认（HITL）

工具执行前检查风险等级，决定是否需要确认：

```
工具调用 →
    风险等级
    ├── safe 🟢       → 直接执行
    ├── confirm 🟡    → 暂停等确认
    │     ├── Web: SSE confirm → 用户点继续/取消
    │     └── CLI: 终端提示 y/N
    └── dangerous 🔴  → 默认拒绝
```

当前风险等级分配：

| 等级 | 工具 |
|------|------|
| 🟢 safe | read_file, list_files, web_search, web_fetch, recall, 等 |
| 🟡 confirm | shell, write_file, self_edit, self_commit |
| 🔴 dangerous | 预留 |

## 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 编程语言 | Python 3.10+ async/await | 快速迭代，生态丰富 |
| LLM 协议 | OpenAI 兼容 API | 可替换多种模型 |
| 工具定义 | JSON Schema | LLM 原生理解 |
| 记忆存储 | SQLite + FTS5 + 向量 | 零配置，多层搜索 |
| UI 方案 | FastAPI + SSE | 实时流，轻量 |
| 子代理 | 独立 Agent 实例 | 上下文隔离 |
| 技能格式 | YAML | 人类可读写，版本可控 |
| 角色配置 | YAML Profile | 零代码新增角色 |
| 风险控制 | per-tool risk_level | 灵活精细的权限管理 |
| 上下文管理 | LLM 摘要压缩 | 突破窗口限制 |
